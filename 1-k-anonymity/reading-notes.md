# 1. Introduction

隐私的定义:  国家机密、商业机密、公民个人数据

隐私泄露的原因:  数据持有者独立操作数据，但是掌握的知识有限，无法控制哪些信息的发布会造成隐私泄露

身份定位: 人口统计数据实验结果表明，87%的公民身份可以通过{邮编、性别、出生日期}等关键属性定位出来


链接攻击: 通过已有信息，链接到外部数据，进而推理出真实身份

隐私泄露案例:

马萨诸塞州 GIC 对外公布了一组医疗数据，其中包括病人的5位邮编、性别、生日等。州长 Weld 对公众保证这份数据是匿名的，因为标识特征 ( 如名字地址 ) 已经被移除。作者通过结合这份数据与她花了20美元购买的选民登记数据，找到了 Weld 的医疗记录。

# 2. Background

2.1 Statistical databases :  联邦州统计部门发布有关民众的统计数据，与其他数据持有者一样，统计部门也面临着对特定个人数据的巨大需求，用于数据挖掘、成本分析、欺诈检测和回顾性研究等应用。 
传统的统计数据库以及数据处理方法(比如添加噪声)已经不再适用。

2.2 Multi-level databases :  数据存储在不同的安全分类中，用户具有不同的安全权限。精确消除所有有关个人身份推理信息的个人数据几乎是不可能的，因为数据持有者不能预先考虑所有可能的攻击。
抑制敏感数据的发布会降低数据质量，数据实用价值降低。


2.3 Computer security is not privacy protection  :  访问控制和身份验证是与计算机安全的传统领域，确保信息接收者有权接收该信息，但是不关注数据发布之后的推理攻击。
本论文工作的前提是发布信息，目的是保护数据主体的身份。

# 3. Methods

Attributes:  一般是个体的唯一标识，比如说姓名、地址、电话等等，这些内容需要在公开数据的时候删掉 

Quasi-identifier(QI):  类似邮编、年龄、生日、性别等不是唯一的，但是能帮助研究人员关联相关数据的标识字段

Sensitive attributes:  敏感数据，比如说患病情况、个人收入等等，这些数据一般是数据研究者最关心的

Inference:  推理就是在其他信息的基础上相信一个新的事实

Disclosure: 有意或者无意的造成个人的特征数据泄露

K匿名的定义:  k-anonymity 的目的是保证公开的数据中包含的个人信息至少 k-1 条不能通过其他个人信息确定出来。也就是公开数据中的任意 quasi-identifier信息，相同的组合都需要出现至少 k 次。



# 4. Attacks against k-anonymity


4.1 unsorted matching attack(无排序攻击)


当公开的数据记录和原始记录的顺序一样的时候，攻击者可以猜出匿名化的记录是属于谁。
解决方法:  在公开数据之前先打乱原始数据的顺序。

4.2 Complementary release attack (补充数据攻击)

如果敏感属性在同一类 quasi-identifiers 中缺乏多样性，或者攻击者有其它的背景知识，k-anonymity 也无法避免隐私泄露


4.3 temporal inference attack(时间推理攻击)

数据集是动态变化的，随着时间的推移已发布的通用数据会受到时间推理攻击。
